# Bible — 安全扫描工作流

> 安全扫描专家必须遵守的检查流程、工具选择规则和交付标准。

## 通用流程

```
理解扫描范围 -> 依赖与配置检查 -> 静态代码审计 -> API 端点审查 -> 认证授权验证 -> 敏感信息扫描 -> 汇总报告 -> 输出修复优先级
```

## 检查清单

### 1. 依赖与供应链安全
- [ ] `npm audit` 检查已知 CVE
- [ ] 检查 package.json 中是否有过时或废弃的依赖
- [ ] 检查 lock 文件完整性
- [ ] 评估关键依赖的安全记录（auth 库、ORM、文件处理库）

### 2. 认证与授权
- [ ] 认证机制完整性（密码哈希算法、盐值、比较方式）
- [ ] 会话管理（JWT 配置、过期策略、刷新机制）
- [ ] 路由保护中间件覆盖所有受保护页面和 API
- [ ] API 端点逐一验证权限检查
- [ ] CSRF 防护措施

### 3. 输入验证与注入防护
- [ ] 所有用户输入经过服务端校验（Zod schema 覆盖）
- [ ] SQL 注入防护（ORM 参数化查询 vs 原始 SQL）
- [ ] XSS 防护（输出编码、Content-Security-Policy）
- [ ] 文件上传验证（MIME 类型、magic bytes、文件大小限制）
- [ ] 路径遍历防护（文件路径消毒）

### 4. 敏感数据保护
- [ ] 环境变量中无硬编码密钥提交到代码库
- [ ] `.env` 文件在 `.gitignore` 中
- [ ] 密码使用 bcrypt/argon2 等慢哈希算法
- [ ] 密钥比较使用 timing-safe 函数
- [ ] 错误响应不泄露内部信息（堆栈、SQL、路径）
- [ ] 日志中无敏感信息明文记录

### 5. API 安全
- [ ] 所有受保护端点返回 401（未认证）或 403（未授权）
- [ ] Webhook/callback 端点验证签名或共享密钥
- [ ] 速率限制或防滥用措施
- [ ] 响应体不包含多余内部信息
- [ ] CORS 配置合理

### 6. 配置与部署安全
- [ ] `.gitignore` 覆盖敏感文件（.env, DB files, uploads, generated code）
- [ ] Next.js 安全相关配置（security headers, CSP）
- [ ] 生产环境无 debug 模式或详细错误
- [ ] HTTPS 强制（生产环境）

## 场景路由

| 场景 | 优先动作 | 关键要求 |
|---|---|---|
| 新功能代码审查 | 按检查清单逐项扫描 | 覆盖新增文件和修改文件 |
| 发布前安全门禁 | 聚焦 CRITICAL/HIGH 问题 | 给出 GO/NO-GO 建议 |
| 依赖漏洞响应 | 运行 npm audit + 评估影响 | 区分可利用 vs 理论风险 |
| 定期安全巡检 | 全量扫描 + 与上次对比 | 标注新增/已修复/残余问题 |
| 事件响应（疑似泄露） | 检查密钥暴露 + 影响评估 | 给出轮换和缓解步骤 |

## 风险等级定义

| 等级 | 定义 | 修复时限 |
|---|---|---|
| **CRITICAL** | 可直接被利用导致数据泄露或系统控制 | 立即修复，阻断发布 |
| **HIGH** | 可被利用但需要特定条件（如已认证用户） | 24 小时内修复 |
| **MEDIUM** | 需要多个条件组合才能利用，或影响有限 | 下一个迭代修复 |
| **LOW** | 理论风险，实际利用概率极低 | 排入技术债务 |
| **INFO** | 安全最佳实践建议，非漏洞 | 可选改进 |

## 交付标准

1. 安全报告必须包含：扫描范围、检查清单覆盖情况、发现列表（含风险等级和位置）、修复建议、残余风险声明。
2. 每个 CRITICAL/HIGH 发现必须说明：触发条件、影响范围、修复代码示例或方向。
3. 报告末尾必须包含「扫描未覆盖区域」声明——未检查的维度不能遗漏。
4. 高影响动作（如建议撤销密钥、暂停服务）需要用户确认。
5. 如果扫描未完成，说明阻塞原因和可继续推进的路径。

## 工具选择

| 检查类型 | 推荐工具/方法 |
|---|---|
| 依赖漏洞 | `npm audit`, `npx audit-ci` |
| 静态代码分析 | 代码模式搜索（Grep/glob）, ESLint security plugin |
| 密钥泄露检测 | `.env` 文件检查, git history scan (`git log -p` 搜索密钥模式) |
| API 安全 | 手动审查 + curl 测试 |
| 配置检查 | 文件审查（next.config, middleware, .gitignore） |
